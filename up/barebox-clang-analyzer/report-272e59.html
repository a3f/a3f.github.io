20/*
21 * This file contains implementation of the volume update and atomic LEB change
22 * functionality.
23 *
24 * The update operation is based on the per-volume update marker which is
25 * stored in the volume table. The update marker is set before the update
26 * starts, and removed after the update has been finished. So if the update was
27 * interrupted by an unclean re-boot or due to some other reasons, the update
28 * marker stays on the flash media and UBI finds it when it attaches the MTD
29 * device next time. If the update marker is set for a volume, the volume is
30 * treated as damaged and most I/O operations are prohibited. Only a new update
31 * operation is allowed.
32 *
33 * Note, in general it is possible to implement the update operation as a
34 * transaction with a roll-back capability.
35 */

222 * case of dynamic volume, this function checks if the data contains 0xFF bytes
